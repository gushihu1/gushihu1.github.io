1. **伪类（Pseudo-classes）的本质：选状态。** 它用于选中 DOM 树中已存在的元素在**某种特定状态或结构特异性**下的样式。它没有创造新元素，只是在既有元素上根据行为或逻辑“动态加上了一个特殊的 class”。
2. **伪元素（Pseudo-elements）的本质：创实体。** 它创造了一个**不在原生物理 DOM 树中、但在渲染树（Render Tree）里真实存在**的“虚拟新抽象元素”。它的行为等同于你人肉向 HTML 内部注入了一个独立的实体标签
3. **CSS1 和 CSS2 时代**：当时并未对这两者做严格的符号区分，伪类和伪元素统一采用单冒号（`:`）书写
4. **CSS3 至今的现代规范**：为了彻底厘清两者的底色差异，W3C 规范明确硬性规定：
   - **单冒号（`:`）**：专门用于**伪类**。
   - **双冒号（`::`）**：专门用于**伪元素**。
