1. static
   1. 设置 `top`、`bottom`、`left`、`right` 以及 `z-index` 属性**完全无效**。
2. relative
   1. **底层机制**：元素**不脱离文档流**，它在标准流中所占有的**物理原始位置（占位）仍然被死死保留**。
   2. 最核心的作用是作为绝对定位的父元素
3. absolute
   1. **底层机制**：元素**完全脱离文档流**，在原本的流中不再占用的空间
   2. 相对于第一个非 `static` 定位的祖先元素（包含块）进行定位。
   3. **渲染性能**：高频动画（如弹幕、飞入特效）首选 `absolute`，因为脱离文档流后，它的几何形变**只会引发局部图层的重绘与回流**，彻底切断了对全局 DOM 树的连锁性能冲击。
4. fixed
   1. **底层机制**：元素**完全脱离文档流**。
   2. **位移参照**：默认严格相对于浏览器视口（Viewport）进行定位。无论页面如何上下滚动，它始终固定在屏幕的特定位置。
   3. **致命跳坑点**：根据 W3C 规范，如果 `fixed` 元素的任意一级父元素设置了 **`transform`、`perspective` 或 `filter` 属性（非 none）**，固定定位的参照物会**瞬间塌陷**，不再相对于视口，而是**降级转为相对于该特殊父元素**
5. sticky
   1. **底层机制**：`relative` 与 `fixed` 的**复合动态状态机**。
   2. **表现行为**：在特定阈值前，它表现为普通的 `relative`，随着页面滚动，当到达指定的边界（如 `top: 10px`）时，它会**瞬间锁定并激活 `fixed` 行为**，形成吸顶/吸底效果。
