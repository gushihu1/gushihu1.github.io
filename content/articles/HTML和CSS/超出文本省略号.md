1.  单行文本截断

   1. ```css
      .single-ellipsis {
        width: 200px;         /* 1. 必须限制盒子的物理宽度或最大宽度占位 */
        white-space: nowrap;  /* 2. 核心：强制文本在一行内连续排布，禁止自动换行 */
        overflow: hidden;     /* 3. 核心：裁剪掉超出盒子物理边界的部分 */
        text-overflow: ellipsis; /* 4. 核心：告诉排版引擎溢出部分用省略号置换 */
      }
      ```

2. 多行文本截断 —— 基于 Webkit 内核的老牌方案

   1. ```css
      .multi-ellipsis-legacy {
        width: 200px;
        overflow: hidden;              /* 1. 裁剪溢出边界的行 */
        text-overflow: ellipsis;       /* 2. 开启省略号置换 */
        display: -webkit-box;          /* 3. 核心：强开旧版 Webkit 弹性伸缩盒模型 */
        -webkit-line-clamp: 3;         /* 4. 核心：限制文本只渲染 3 行，多余的在第3行末尾打上省略号 */
        -webkit-box-orient: vertical;  /* 5. 核心：指定伸缩盒子的子元素纵向排列 */
      }
      ```

3. 现代纯 CSS 规范方案：原生 `line-clamp` 升级

   1. ```css
      .multi-ellipsis-modern {
        width: 200px;
        /* 现代标准规范：无需开启 -webkit-box 这种历史古董盒模型，直接在标准流下一行控高 */
        line-clamp: 3;
        overflow: hidden; 
      }
      ```

4. 经典痛点

   1. 在 Flex 弹性布局中省略号无故失效
      1. Flex 项目的默认最小宽度属性 `min-width` 的隐式值是 `auto`，而非 0。当遭遇长文本时，`auto` 机制为了保证内容的完整，会强制要求项目必须横向撑开以容纳下文本的物理长度。
      2. 必须打破浏览器的 `auto` 局限。直接在那个被挤爆的 `flex: 1` 的右侧大盒子上加上一行 **`min-width: 0;`**
   2. 多行省略在遭遇 `line-height` 或 `padding` 时引发的“半截文字”
      1. 在使用多行省略的容器上，**严格禁止硬编码写死固定的 `height`**，尺寸必须交给内容动态收缩
      2. 改为在父级或兄弟节点上使用 **`margin` 或现代标准的 `gap` 属性**进行物理隔离

5. 只有超出时才显示 Tooltip 提示

   1. 动态读取该文本节点的物理滚动宽度（`scrollWidth`）与当前的视觉边界可感知宽度（`clientWidth`）。如果 **`scrollWidth > clientWidth`**，说明此时文本在标准流中已经发生了物理溢出并被省略号隐藏了
