1. **语义化（有状态树）**：像 `<article>`、`<nav>`、`<header>`，它们通过标签名**明确声明了其内部包裹物料的“角色（Role）”与“职责（Responsibility）”**
2. 核心语义化骨架与标准布局模型
   1. **`<header>`**：定义页面、组件或特定分区的头部导流区。
   2. **`<nav>`**：定义核心导航链接组。**面试避坑点**：不要把页面上所有的 `<a>` 标签都用 `nav` 包裹，它严格只用于整站的主导航、面包屑或核心分页栏。
   3. **`<main>`**：定义文档的**主体核心内容区**。一个 HTML 文档内**有且只能出现一个**处于可见状态的 `<main>` 标签，其内部不应包含全站共用的侧边栏、页脚或版权声明。
   4. **`<article>`**：定义一段**完全独立且高内聚的完整内容**。它的黄金判定准则是：如果把这段内容单独抽离出来转发到另一个网站，它依然具备完整的可读性和语境（如：博客文章、新闻条目、用户独立评论卡片）。
   5. **`<section>`**：定义文档中的一个**逻辑章节或主题分组**。它通常应当包含一个标题（`<h1>-<h6>`）。如果一个盒子纯粹是为了加样式或写个轮播图，严格禁止使用 `section`（应退回使用 `div`）。
   6. **`<aside>`**：定义与主体内容间接相关的附属信息流（如：博客侧边侧边栏、词条解释、右侧高频热点推荐）。
   7. **`<footer>`**：定义页面或特定分区底部的版权、友情链接栏。
3. 功能
   1. 当浏览器解析 HTML 时，它不仅会构建我们熟知的 **DOM 树**，还会并发构建一个供操作系统、弱视人群及屏幕阅读器（Screen Readers）读取的 **无障碍树（Accessibility Tree）**，天然达成 WAI-ARIA 国际无障碍规范。
   2. **流量获客与 SEO 数据工程中**，语义化标签能够主动降低搜索引擎爬虫的语义解算开销，为核心业务文本在爬虫的倒排索引模型中争取到了最高的权重加权，实现了显著的工程降本增效
