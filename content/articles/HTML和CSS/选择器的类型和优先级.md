1. **基本选择器**：

   - 通配选择器（`*`）
   - 元素/标签选择器（如 `div`、`p`）
   - 类选择器（如 `.active`、`.nav`）
   - ID 选择器（如 `#app`、`#header`）
   - 属性选择器（如 `[type="text"]`、`[disabled]`）
   - important

2. **组合/关系选择器**：

   - 后代选择器（`空格`，如 `.box p`）
   - 子代选择器（`>`，如 `.box > p`）
   - 相邻兄弟选择器（`+`，如 `.item + .item`，仅匹配紧跟在后面的**一个**兄弟）
   - 通用兄弟选择器（`~`，如 `.item ~ .item`，匹配后面**所有**符合条件的兄弟）

3. **伪类选择器（Pseudo-classes，用于表示处于某种特定状态的元素）**：

   - 动态状态：`:hover`、`:focus`、`:active`
   - 结构伪类：`:nth-child(n)`、`:first-child`、`:not(.disabled)`
   - *高级新特性（面试加分点）*：**`:has()`**（父选择器/结构先知，如 `.card:has(.active)`）、`:where()` 和 `:is()`（常用于精简代码与控制权重）。

4. **伪元素选择器（Pseudo-elements，用于创建或选中虚拟的文档片段）**：

   - `::before`、`::after`（必须配合 `content` 属性使用）
   - `::first-line`、`::placeholder`、`::selection`

5. | **级别**                | **代表的计数位**   |
   | ----------------------- | ------------------ |
   | **第一位 (1, 0, 0, 0)** | **行内样式**       |
   | **第二位 (0, 1, 0, 0)** | **ID 选择器**      |
   | **第三位 (0, 0, 1, 0)** | **类、属性、伪类** |
   | **第四位 (0, 0, 0, 1)** | **标签、伪元素**   |

6. **忽略项（0, 0, 0, 0）**：通配符（`*`）、关系连接符（`+`、`>`、`~`）以及**伪类 `:not()`、`:is()` 自身**不占用任何权重。但是，写在 `:not()` 或 `:is()` **括号内部的选择器会正常计算权重**。

7. 精准比对逻辑（大厂最爱纠正的误区）

   1. **规则 1：从左往右，逐位比对**。浏览器计算出两个选择器的四元组后，会先对比第一位，大者胜出；若第一位相同，再对比第二位，以此类推。
   2. **规则 2：不存在所谓的“10个类选择器干掉1个ID选择器”**。

8. 它不属于特异性四元组，但一旦属性后面加上了 `!important`，其权重直接提升至最高级。

9. **后来者居上原则**：若权重完全相同，**在 CSS 样式表中物理位置靠后的规则会覆盖掉靠前的规则**。

10. **继承属性的权重最低**： 元素从父级继承来的样式（如 `color`、`font-size`），其权重低于任何直接指向该元素的选择器（即使该选择器只是一个通配符 `*`）。

11. **前沿降权机制：`:where()` 伪类**： 现代 CSS 引入了 `:where()` 伪类（如 `:where(.class-a, .class-b)`）。**它的核心特性是其内部的所有选择器权重直接归零**。
