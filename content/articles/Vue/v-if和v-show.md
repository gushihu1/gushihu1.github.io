| **维度**             | **v-if (条件渲染)**                                          | **v-show (条件显示)**                                        |
| -------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **底层实现本质**     | **运行时动态控制 VNode。** 编译出的 Render 函数会包含条件条件条件条件三元表达式。 | **动态修改运行时样式。** 无论条件条件条件条件是否满足，均会无脑生成 VNode 并挂载。 |
| **DOM 节点状态**     | 为 `false` 时，真实 DOM 树中**完全没有**该节点（或仅保留一个 `` 占位符注释节点）。 | 为 `false` 时，DOM 节点**依然存在**，只是被强行注入了 `display: none !important;` 样式。 |
| **浏览器渲染流水线** | **触发回流（Reflow）与重绘（Repaint）。** 节点的插入或注销会破坏全量 DOM 几何尺寸。 | **通常仅触发合成或最小重绘。** `display: none` 切换会使节点脱离文档流，不引发子孙重绘。 |
| **生命周期触发**     | 状态切换时，内部子组件会**完整经历**创建与注销链路（`beforeCreate` > `mounted` / `unmounted`）。 | **只在初次挂载时触发一次生命周期**，后续状态变更**绝对不触发**任何生命周期钩子。 |
| **开销平摊模型**     | **更高的切换开销**（涉及虚拟 DOM Tree 的 Diff、垃圾回收 GC 压力、DOM 对象的频繁创建与销毁）。 | **更高的初始渲染开销**（不论用不用，初次加载时全量组件和节点都会在内存中进行初始化与挂载）。 |
