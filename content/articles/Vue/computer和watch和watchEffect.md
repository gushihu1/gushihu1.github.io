| **维度**            | **computed (计算属性)**                                      | **watch (监听器)**                                           |
| ------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **底层核心标志**    | **缓存标记位：`Dirty Flag`（脏检查）。**                     | **副作用任务队列：`Scheduler`调度器。**                      |
| **数据更新时机**    | **惰性求值（Lazy Evaluation）。** 只有当依赖的属性改变，*且*外部有业务真实读取了它的 `.value` 时，才会重新计算。 | **即时响应或异步批处理。** 监听到数据变化，立刻将回调函数推入微任务队列（默认）。 |
| **缓存机制开销**    | **有。** 内部持有一个 `_value` 缓存值。只要 `dirty` 标志位为 `false`，无脑直接返回缓存，**执行开销为 $O(1)$**。 | **无。** 每次数据发生变动，只要满足条件，对应的回调函数**必定会被 100% 触发执行**。 |
| **异步/副作用支持** | **坚决禁止。** 内部必须是纯函数（Pure Function），绝对不能包含任何 `setTimeout` 或 `fetch`。 | **天然契合。** 天生就是为了执行网络拉取、手动修改 DOM 或全局埋点等异步副作用而生的。 |
| **返回值特征**      | **必须有返回值。** 产生一个全新的只读响应式对象。            | **绝对不需要返回值。** 它只负责执行动作（Action）。          |

| **维度**         | **watch (侦听器)**                                           | **watchEffect (高级副作用监听)**                             |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **依赖声明机制** | **显式指定路径。** 必须明确指出监听哪一个或哪一组数据源（`watch(source, cb)`）。 | **隐式自动依赖收集。** 内部执行到谁就监听谁，不需要手动维护依赖数组。 |
| **初次执行时机** | 默认**惰性执行（Lazy）**，只有当监听的值变了才触发（可配置 `immediate: true`）。 | **立即同步执行一次（Immediate）。** 必须先运行一次以收集首次依赖项。 |
| **新旧数据访问** | **可以**在回调中完美拿到 `newValue` 和 `oldValue`。          | **无法**直接拿到旧值，它聚焦于“当前状态与外部副作用的映射”。 |
| **深度监听开销** | 监听大对象时需要配置 `deep: true`，底层会经历昂贵的递归变量扫描。 | 不需要 deep，**只追踪函数同步执行过程中读取到的属性**。      |

1. **只要需要产生一个“新状态 / 新变量”**，且这个变量依赖其他变量计算而来 > **100% 选用 `computed`**。
2. **只要是状态变了，需要执行“一步具体的动作”**（如网络请求、定时器、手动 DOM、跨页面跳转），且需要**对比新旧值（`oldValue`）**  **100% 选用 `watch`**。
3. **只要是初始化时必须立刻执行一次**，且需要**自动收集依赖、动态监听多个变量去同步某种外部副作用**（如 LocalStorage 写入、WebSocket 状态栈维护）  **100% 选用 `watchEffect`**。
