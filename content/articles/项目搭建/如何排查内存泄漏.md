1. 内存泄漏是指程序中已动态分配的堆内存由于某种原因未释放或无法释放，造成系统内存的浪费，导致程序运行速度减慢甚至系统崩溃
2. 产生原因
   1. 全局变量
   2. 未清理的计时器
   3. 事件监听
   4. 闭包
   5. 控制台打印
   6. 未销毁的游离DOM
   7. 或者如大屏里面，创建了图表或者打点图标没有进行销毁等等
3. 首先确认是内存泄漏
   1. 打开 Chrome DevTools 的 **Performance** 面板。
   2. 勾选 **Memory** 选项。
   3. 点击录制，在页面上进行频繁的交互操作（如打开/关闭大屏组件、切换路由）。
   4. 停止录制后观察 **JS Heap** 曲线：如果曲线呈现典型的“锯齿状”且**波谷不断抬高**，手动进行回收依然降低不多，说明内存没有被完全释放，存在泄漏风险。
4. 去memory面板定位Heap Snapshot
   1. **拍快照 A**：页面初始状态。
   2. **执行操作**：重复你怀疑有问题的交互。
   3. **拍快照 B**：操作完成后的状态。
   4. **对比视图 (Comparison)**：在快照 B 中选择 "Comparison"，对比 A 和 B。查看 **Constructor** 列中增加的对象。
