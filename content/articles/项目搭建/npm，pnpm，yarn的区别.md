1. 他们三个都是包管理工具，用来管理依赖
2. npm 
   1. node自带的包管理工具，定义了规范，最久远，生态完善
   2. 最开始是嵌套结构
      1. 结构清晰，易于理解
      2. 依赖大量重复，会重复存储下载
   3. 是扁平化策略，把依赖提升到顶层
      1. 解决了重复的问题
      2. 可能会出现幽灵依赖
         1. 就是项目package.josn没有直接声明某个依赖，加入是A
         2. 但是使用的依赖 B 引用了 A
         3. 这时候你可以直接使用这个依赖 A
         4. 如果以后使用的依赖 B 升级或者不再使用 依赖A，就会导致问题
      3. 可能会出现依赖分身
         1. 有两个不同的库依赖两个不同版本的同一个内容
3. yarn
   1. npm 下载是很慢的，
      1. yarn引入了并行下载
      2. 进行了离线缓存的策略，避免了从网络重复下载
   2. 引入了 .lock 文件，确保了不同环境版本号一致
   3. 使用的也是扁平化安装
4. pnpm
   1. 全局只存储一份依赖仓库，用的时候直接把地址指向仓库的地址，不需要复制过来
   2. 直接解决了幽灵依赖，只有声明的依赖才会到顶层可以使用

5. 建议直接使用pnpm
