| **特性**          | **TCP (Transmission Control Protocol)**                      | **UDP (User Datagram Protocol)**                             |
| ----------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **连接状态**      | **面向连接**。传输数据前必须通过“三次握手”建立连接，传输结束后需“四次挥手”释放连接。 | **无连接**。不需要建立连接，知道对方的 IP 和端口直接发送数据。 |
| **可靠性**        | **可靠传输**。通过序列号、确认应答、超时重传、错误校验机制，确保数据**无差错、不丢失、不重复且按序到达**。 | **不可靠传输**。只管发送数据，不保证数据能成功到达，也不保证顺序，可能发生丢包或乱序。 |
| **传输形式**      | **面向字节流**。数据没有边界，TCP 会根据网络状况将数据拆分或合并发送，前端接收时可能需要处理**粘包/拆包**。 | **面向报文**。保留应用层交付的报文边界，一次发送一个完整的报文，不合并也不拆分。 |
| **首部开销**      | 较大。最小 **20 字节**，因为要包含序列号、确认号、窗口大小、控制标志等。 | 较小。固定 **8 字节**，仅包含源端口、目的端口、长度和校验和。 |
| **拥塞/流量控制** | **有**。具备滑动窗口机制（流量控制）以及慢启动、拥塞避免、快重传、快恢复机制（拥塞控制），会根据网络拥堵自动限速。 | **无**。没有控制机制，即使网络极度拥堵，也会以恒定的速度向外发送，不会主动减速。 |
| **通信对象**      | 只能是**点对点**（1 对 1）的单播通信。                       | 支持单播、多播和广播（1 对 1、1 对多、多对多）。             |
