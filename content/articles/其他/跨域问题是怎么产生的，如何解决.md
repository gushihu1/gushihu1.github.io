1. 浏览器的同源策略，同一协议，同一域名，同一端口，只在浏览器中生效
2. 通过后端服务器配置，解决cors问题，Access-Control-Allow-Origin
3. 前端自己进行建立一个代理服务器
4. 后端通过nginx进行反向代理，我们请求到这个代理服务器上再通过服务器转发也能解决
5. JSONP
   1. 浏览器限制了 `XMLHttpRequest` 或 `Fetch` 发生跨域请求，但它允许 `<script>`、`<img>`、`<link>` 等带有 `src` 或 `href` 属性的标签**不受跨域限制**地加载外部资源
   2. JSONP 利用了 `<script>` 标签可以跨域加载 JS 文件的特性。它让前端声明一个回调函数，然后动态创建 `<script>` 标签去请求后端；后端收到请求后，**不返回纯 JSON 数据，而是返回一段 JS 代码，这段代码正是调用了前端的回调函数，并将 JSON 数据作为参数传了进去**
   3. 已经几乎完全淘汰
