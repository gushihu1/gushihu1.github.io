| **维度**          | **GET**                                                      | **POST**                                                     |
| ----------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **RFC 规范语义**  | **安全（Safe）且幂等（Idempotent）**。用于获取/检索资源。    | **非安全且非幂等**。用于创建/修改/处理资源。                 |
| **参数携带方式**  | 只能通过 **URL (Query String)** 传递，遵循 `application/x-www-form-urlencoded` 编码。 | 可以通过 **Request Body** 传递，支持 `json`、`form-data`、`xml`、二进制流等多种 Content-Type。 |
| **数据长度限制**  | HTTP 协议本身无限制，但受到**浏览器和 Web 服务器对 URL 长度的物理限制**（通常为 2KB ~ 8KB）。 | HTTP 协议本身无限制，通常只受服务器配置（如 Nginx 的 `client_max_body_size`）限制。 |
| **缓存机制**      | **可被浏览器、代理服务器（CDN）主动缓存**。支持根据 Cache-Control、ETag 进行协商缓存。 | **默认不被缓存**。除非显式设置特定响应头（极少使用）。       |
| **后退/刷新行为** | 无害。浏览器直接从缓存中读取，或重新发起请求。               | 提示风险。浏览器通常会弹窗提示“是否重新提交表单”，防止重复创建数据。 |
