1. Promise 实质上是一个有限状态机，它解决了传统回调地狱（Callback Hell）带来的控制权反转问题。

2. **三个状态**：

   - `pending`（进行中）：初始状态。
   - `fulfilled`（已成功）：操作成功完成。
   - `rejected`（已失败）：操作失败。

3. **两个特点**：

   1. **状态不可逆**：一旦从 `pending` 变为 `fulfilled` 或 `rejected`，状态就凝固了，任何时候都可以获取结果。
   2. **结果锁定**：如果状态改变后再添加回调，会立即得到结果。

4. | **方法**                 | **核心逻辑**                           | **典型应用场景**                           |
   | ------------------------ | -------------------------------------- | ------------------------------------------ |
   | **`Promise.all`**        | 全部成功才成功，有一个失败就立刻失败。 | 多个并行的初始化接口全部完成后才加载页面。 |
   | **`Promise.allSettled`** | 无论成功失败，返回所有结果的数组。     | 批量上传文件，需要知道哪些成功哪些失败。   |
   | **`Promise.race`**       | 谁跑得快就用谁的结果（无论成功失败）。 | 请求超时控制（接口请求 vs 定时器）。       |
   | **`Promise.any`**        | 只要有一个成功就成功，全失败才失败。   | 备用服务器调度（尝试从多个节点获取资源）。 |

5. **误区**：很多人认为 `.then()` 返回的是原来的 Promise。

   **真相**：`.then()` **每次都会返回一个新的 Promise 实例**。

   - 如果回调返回一个值，新 Promise 会 resolve 这个值。
   - 如果回调抛出异常，新 Promise 会 reject。
   - 如果回调返回另一个 Promise，新 Promise 会“跟随”那个 Promise 的状态。
